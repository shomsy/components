ServiceLifetime

Enum defining how long a service instance lives.

Foundation/Container/Features/Core/Enum/ServiceLifetime.php

Enum Overview

Specifies the persistence policy for service instances in the container.

More

Three lifetime options:

  • Singleton: One shared instance for the entire application.
  • Scoped: One instance per scope (e.g., per HTTP request).
  • Transient: New instance every time it's requested.

This enum ensures type-safe configuration and makes the container's caching behavior explicit.

Story Examples

Choosing the Right Lifetime

Singleton - Database Connection Pool

$container->bind(DatabasePool::class)
    ->lifetime(ServiceLifetime::Singleton);

// Same instance everywhere
$pool1 = $container->get(DatabasePool::class);
$pool2 = $container->get(DatabasePool::class);
// $pool1 === $pool2 ✓

Use when: Service is stateless or manages global state (config, connection pools, caches).

Scoped - HTTP Request Context

$container->bind(RequestContext::class)
    ->lifetime(ServiceLifetime::Scoped);

$container->beginScope(); // Start of HTTP request
$ctx1 = $container->get(RequestContext::class);
$ctx2 = $container->get(RequestContext::class);
// $ctx1 === $ctx2 ✓ (same request)

$container->endScope(); // End of request
$container->beginScope(); // New request
$ctx3 = $container->get(RequestContext::class);
// $ctx3 !== $ctx1 ✓ (different request)

Use when: Service holds request-specific state (user session, request data, transaction).

Transient - Command Objects

$container->bind(SendEmailCommand::class)
    ->lifetime(ServiceLifetime::Transient);

$cmd1 = $container->get(SendEmailCommand::class);
$cmd2 = $container->get(SendEmailCommand::class);
// $cmd1 !== $cmd2 ✓ (always fresh)

Use when: Service is lightweight and needs fresh state (commands, DTOs, value objects).

Performance Considerations
  • Singleton: Fastest (resolved once, cached forever).
  • Scoped: Fast within scope (resolved once per request).
  • Transient: Slowest (resolved every time, no caching).
