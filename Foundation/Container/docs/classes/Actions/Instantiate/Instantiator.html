Instantiator

Creates new instances of classes with automatic constructor dependency injection.

Foundation/Container/Features/Actions/Instantiate/Instantiator.php

Quick Summary

Instantiator is responsible for the "new" operation in dependency injection. It analyzes constructor parameters, resolves dependencies, and creates fresh instances with all dependencies injected.

Story Example: Creating a UserService

Scenario: Automatic Constructor Injection

You have a UserService that needs a repository and logger:

class UserService
{
    public function __construct(
        private UserRepository $repository,
        private Logger $logger
    ) {}
    
    public function createUser(array $data): User
    {
        $this->logger->info('Creating user', $data);
        return $this->repository->create($data);
    }
}

// Container automatically resolves dependencies
$userService = $container->make(UserService::class);
// Instantiator created UserService with injected dependencies!
Step-by-Step Process
  1. Get Prototype: ServicePrototypeFactory provides UserService prototype.
  2. Analyze Constructor: Finds 2 parameters: UserRepository and Logger.
  3. Resolve Dependencies: DependencyResolver resolves both from container.
  4. Create Instance: Calls new UserService($repository, $logger).
  5. Return: Fully configured instance ready to use.

Note: This happens before property/method injection by InjectDependencies.

How It Works

Constructor Injection Workflow

Instantiator follows a precise workflow to ensure correct object creation:

1. Validation

Check if class exists and is instantiable

class_exists($class) && $reflection->isInstantiable()

2. Prototype Retrieval

Get cached analysis or analyze class

$prototype = $factory->createFor($class)

3. Parameter Resolution

Resolve all constructor parameters

$params = $resolver->resolveParameters(...)

4. Instantiation

Create instance with resolved parameters

$reflection->newInstanceArgs($params)
Handling Edge Cases

No Constructor

class SimpleService
{
    // No constructor
}

// Instantiator calls: new SimpleService()

Constructor with Overrides

$service = $container->make(UserService::class, [
    'logger' => new NullLogger() // Override specific dependency
]);

Non-Instantiable Classes

abstract class AbstractService {}

// Throws: ContainerException
// "Cannot autowire: class [AbstractService] is not instantiable."

Architecture

Role in Resolution Pipeline

Instantiator is the "Build" step in the container's resolution pipeline:

1. ResolutionEngine

Orchestrates resolution

→
2. Instantiator

Creates instance

→
3. InjectDependencies

Property/method injection

→
4. Return

Fully configured service

Performance Considerations

Optimization Strategies

Caching & Performance
  • Prototype Caching: Class analysis is cached, avoiding repeated reflection.
  • Lazy Resolution: Dependencies are only resolved when needed.
  • Context Passing: Metadata is passed through context to avoid re-analysis.
// First call: Analyzes UserService
$user1 = $container->make(UserService::class);

// Subsequent calls: Uses cached prototype (fast!)
