ResolutionContext

Immutable state holder for a single resolution request.

Foundation/Container/Act/Resolve/ResolutionContext.php

Class Overview

ResolutionContext carries the who/what/where of a resolution. It stores the service name, overrides, depth, and parent so the engine can detect circular dependencies and provide accurate error messages.

More

Imagine a breadcrumb trail. Each time the container resolves a dependency, it creates a child context pointing back to its parent. If a service tries to resolve itself again, the context can detect that loop.

Methods

__construct

Creates a new resolution context with service name, overrides, and lineage.

More

The context is immutable: once created, it does not change. New child contexts are created for nested resolutions.

withEngine

Returns a copy of the context with the resolution engine attached.

More

This is useful when recursive resolution needs a reference to the engine.

withPrototype

Returns a copy of the context with a precomputed prototype attached.

More

When a prototype is available, the engine can skip extra analysis steps.

child

Creates a child context for a nested dependency.

More

Every nested dependency gets its own context so the engine can track depth and prevent loops.

getAbstract

Returns the abstract identifier being resolved.

getOverrides

Returns explicit parameter overrides supplied by the caller.

getDepth

Returns the depth of this resolution within the dependency graph.

getParent

Returns the parent context, or null if this is the root.

getEngine

Returns the attached engine instance, if present.

getPrototype

Returns the attached prototype, if present.

contains

Checks whether the resolution path already contains a given identifier.

More

This is used to detect circular dependencies like A → B → A.

getPath

Returns a string path of the resolution chain.

More

Useful for debugging: it shows the exact order of dependencies leading to a failure.
