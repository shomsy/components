LazyValue

Defers expensive service creation until the value is actually needed.

Foundation/Container/Features/Actions/Advanced/Lazy/LazyValue.php

Quick Summary

LazyValue wraps a factory closure and only executes it when get() is called for the first time. This enables lazy loading of expensive services, improving application startup time.

Story Example: Lazy Database Connection

Scenario: Expensive Service That's Rarely Used

You have a ReportGenerator that needs a database connection, but reports are only generated occasionally:

use Avax\Container\Features\Actions\Advanced\Lazy\LazyValue;

// Without LazyValue - connects immediately (slow startup!)
$db = new DatabaseConnection('host', 'user', 'pass');
$generator = new ReportGenerator($db);

// With LazyValue - connects only when needed (fast startup!)
$lazyDb = new LazyValue(fn() => new DatabaseConnection('host', 'user', 'pass'));
$generator = new ReportGenerator($lazyDb);

// No connection yet! ✓

// Later, when report is actually generated:
$report = $generator->generate();
// NOW the connection is created! ✓
How It Works Internally
  1. Construction: Store the factory closure, don't execute it.
  2. First get(): Execute factory, cache result, mark as initialized.
  3. Subsequent get(): Return cached value immediately (no re-execution).
  4. Exception Safety: If factory throws, mark as initialized to prevent retry loops.
$lazy = new LazyValue(fn() => expensiveOperation());

// First call - executes factory
$value1 = $lazy->get(); // expensiveOperation() runs

// Second call - returns cached value
$value2 = $lazy->get(); // No execution, returns cached

// $value1 === $value2 ✓

How It Works

Lazy Initialization Pattern

LazyValue implements the classic Lazy Initialization pattern with thread-safe execution guarantees:

State: Uninitialized

$initialized = false

$value = null

↓ get() called

Execute Factory

$value = ($factory)()

Mark $initialized = true

↓ subsequent get()

Return Cached

Skip factory execution

Return $value immediately

Edge Cases & Error Handling

Factory Throws Exception

$lazy = new LazyValue(fn() => throw new RuntimeException('Failed!'));

try {
    $lazy->get(); // Throws RuntimeException
} catch (RuntimeException $e) {
    // Exception propagates
}

// Second call - STILL throws (factory not re-executed)
$lazy->get(); // Returns null (marked initialized)

Factory Returns Null

$lazy = new LazyValue(fn() => null);
$value = $lazy->get(); // null is valid!

Container Integration

// Container can inject LazyValue
class Service
{
    public function __construct(
        private LazyValue $expensiveService
    ) {}
    
    public function doWork()
    {
        // Only resolve when needed
        $service = $this->expensiveService->get();
        $service->process();
    }
}

Architecture

Role in Container Ecosystem

LazyValue is an optional optimization for expensive services:

1. Service Definition

Wrap factory in LazyValue

→
2. Injection

LazyValue injected (fast!)

→
3. First Use

Factory executes on get()

→
4. Cached

Subsequent calls instant

When to Use LazyValue

✓ Good Use Cases:

  • Database connections that aren't always needed
  • External API clients (HTTP, SOAP, etc.)
  • Heavy computation or file I/O
  • Services with expensive constructors

✗ Avoid For:

  • Lightweight objects (overhead not worth it)
  • Services always used (no benefit)
  • Critical path operations (adds indirection)

Benchmarks & Optimization

Performance Characteristics
Operation Cost Notes
Construction ~0.001ms Just stores closure
First get() Factory cost + ~0.002ms Executes factory once
Subsequent get() ~0.0001ms Returns cached value
